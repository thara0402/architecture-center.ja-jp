## <a name="why-partition-data"></a><span data-ttu-id="f8a35-111">データをパーティション分割する理由</span><span class="sxs-lookup"><span data-stu-id="f8a35-111">Why partition data?</span></span>

* <span data-ttu-id="f8a35-112">**拡張性の向上**。</span><span class="sxs-lookup"><span data-stu-id="f8a35-112">**Improve scalability**.</span></span> <span data-ttu-id="f8a35-113">単一のデータベース システムをスケールアップすると、最終的には物理的なハードウェア限界に到達します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-113">When you scale up a single database system, it will eventually reach a physical hardware limit.</span></span> <span data-ttu-id="f8a35-114">データを複数のパーティションに分割し、各パーティションを個別のサーバー上でホストすると、システムをほぼ無制限にスケールアウトできます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-114">If you divide data across multiple partitions, each hosted on a separate server, you can scale out the system almost indefinitely.</span></span>

* <span data-ttu-id="f8a35-115">**パフォーマンスの向上**。</span><span class="sxs-lookup"><span data-stu-id="f8a35-115">**Improve performance**.</span></span> <span data-ttu-id="f8a35-116">各パーティションでのデータ アクセス操作は、より少量のデータに対して実行されます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-116">Data access operations on each partition take place over a smaller volume of data.</span></span> <span data-ttu-id="f8a35-117">パーティション分割を適切に行うことで、システムの効率を高めることができます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-117">Correctly done, partitioning can make your system more efficient.</span></span> <span data-ttu-id="f8a35-118">複数のパーティションに影響する操作は、並列に実行できます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-118">Operations that affect more than one partition can run in parallel.</span></span>

* <span data-ttu-id="f8a35-119">**セキュリティの向上**。</span><span class="sxs-lookup"><span data-stu-id="f8a35-119">**Improve security**.</span></span> <span data-ttu-id="f8a35-120">場合によっては、機密データとそれ以外のデータを異なるパーティションに分け、機密データにさまざまなセキュリティ制御を適用できます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-120">In some cases, you can separate sensitive and non-sensitive data into different partitions and apply different security controls to the sensitive data.</span></span>

* <span data-ttu-id="f8a35-121">**運用上の柔軟性の向上**。</span><span class="sxs-lookup"><span data-stu-id="f8a35-121">**Provide operational flexibility**.</span></span> <span data-ttu-id="f8a35-122">パーティション分割は、運用の微調整、管理効率の最大化、およびコストの最小化を達成するための多くの機会を提供します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-122">Partitioning offers many opportunities for fine tuning operations, maximizing administrative efficiency, and minimizing cost.</span></span> <span data-ttu-id="f8a35-123">たとえば、各パーティションのデータの重要性に基づいて、管理、監視、バックアップと復元、および他の管理タスクについて、異なる戦略を定義することができます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-123">For example, you can define different strategies for management, monitoring, backup and restore, and other administrative tasks based on the importance of the data in each partition.</span></span>

* <span data-ttu-id="f8a35-124">**データ ストアと使用パターンの一致**。</span><span class="sxs-lookup"><span data-stu-id="f8a35-124">**Match the data store to the pattern of use**.</span></span> <span data-ttu-id="f8a35-125">パーティション分割では、コストとデータ ストアが提供する組み込み機能に基づいて、各パーティションを異なるタイプのデータ ストアにデプロイできます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-125">Partitioning allows each partition to be deployed on a different type of data store, based on cost and the built-in features that data store offers.</span></span> <span data-ttu-id="f8a35-126">たとえば、大量のバイナリ データを Blob Storage に格納し、構造化されたデータをドキュメント データベースに保持することができます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-126">For example, large binary data can be stored in  blob storage, while more structured data can be held in a document database.</span></span> <span data-ttu-id="f8a35-127">「[適切なデータ ストアの選択](../guide/technology-choices/data-store-overview.md)」をご覧ください。</span><span class="sxs-lookup"><span data-stu-id="f8a35-127">See [Choose the right data store](../guide/technology-choices/data-store-overview.md).</span></span>

* <span data-ttu-id="f8a35-128">**可用性の向上**。</span><span class="sxs-lookup"><span data-stu-id="f8a35-128">**Improve availability**.</span></span> <span data-ttu-id="f8a35-129">データを複数のサーバーにまたがって分割することで、単一障害点を避けることができます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-129">Separating data across multiple servers avoids a single point of failure.</span></span> <span data-ttu-id="f8a35-130">1 つのインスタンスで障害が発生した場合、使用できなくなるのは、そのパーティションのデータだけです。</span><span class="sxs-lookup"><span data-stu-id="f8a35-130">If one instance fails, only the data in that partition is unavailable.</span></span> <span data-ttu-id="f8a35-131">その他のパーティションでの操作は、続行できます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-131">Operations on other partitions can continue.</span></span> <span data-ttu-id="f8a35-132">マネージド PaaS データ ストアの場合、この考慮事項はあまり関係がありません。これらのサービスでは、設計に冗長性が組み込まれているためです。</span><span class="sxs-lookup"><span data-stu-id="f8a35-132">For managed PaaS data stores, this consideration is less relevant, because these services are designed with built-in redundancy.</span></span> 

## <a name="designing-partitions"></a><span data-ttu-id="f8a35-133">パーティションの設計</span><span class="sxs-lookup"><span data-stu-id="f8a35-133">Designing partitions</span></span>

<span data-ttu-id="f8a35-134">データをパーティション分割する際の一般的な 3 つの戦略を次に示します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-134">There are three typical strategies for partitioning data:</span></span>

* <span data-ttu-id="f8a35-135">**水平的パーティション分割** (しばしば "*シャーディング*" と呼ばれます)。</span><span class="sxs-lookup"><span data-stu-id="f8a35-135">**Horizontal partitioning** (often called *sharding*).</span></span> <span data-ttu-id="f8a35-136">この戦略では、各パーティションは個別のデータ ストアですが、すべてのパーティションが同じスキーマを持ちます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-136">In this strategy, each partition is a separate data store, but all partitions have the same schema.</span></span> <span data-ttu-id="f8a35-137">各パーティションは "*シャード*" と呼ばれ、データの特定のサブセット (特定の顧客セットのすべての注文など) を保持します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-137">Each partition is known as a *shard* and holds a specific subset of the data, such as all the orders for a specific set of customers.</span></span>
* <span data-ttu-id="f8a35-138">**垂直的パーティション分割**。</span><span class="sxs-lookup"><span data-stu-id="f8a35-138">**Vertical partitioning**.</span></span> <span data-ttu-id="f8a35-139">この戦略では、各パーティションはデータ ストアに含まれる項目のフィールドのサブセットを含みます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-139">In this strategy, each partition holds a subset of the fields for items in the data store.</span></span> <span data-ttu-id="f8a35-140">フィールドは、それらの使用パターンに従って分割されます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-140">The fields are divided according to their pattern of use.</span></span> <span data-ttu-id="f8a35-141">たとえば、頻繁にアクセスされるフィールドを 1 つの垂直的パーティションに、使用頻度の少ないフィールドをまとめて別の垂直的パーティションに配置します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-141">For example, frequently accessed fields might be placed in one vertical partition and less frequently accessed fields in another.</span></span>
* <span data-ttu-id="f8a35-142">**機能的パーティション分割**。</span><span class="sxs-lookup"><span data-stu-id="f8a35-142">**Functional partitioning**.</span></span> <span data-ttu-id="f8a35-143">この戦略では、システム内の区分可能な各コンテキストによって使用される方法に従って、データは集約されます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-143">In this strategy, data is aggregated according to how it is used by each bounded context in the system.</span></span> <span data-ttu-id="f8a35-144">たとえば、e コマース システムでは、請求書データと製品在庫データを、それぞれ異なるパーティションに格納できます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-144">For example, an e-commerce system might store invoice data in one partition and product inventory data in another.</span></span>

<span data-ttu-id="f8a35-145">これらの戦略は組み合わせることができます。パーティション分割構成を設計するときには、これらをすべて検討することをお勧めします。</span><span class="sxs-lookup"><span data-stu-id="f8a35-145">These strategies can be combined, and we recommend that you consider them all when you design a partitioning scheme.</span></span> <span data-ttu-id="f8a35-146">たとえば、データをシャードに分割し、次に垂直的パーティション分割を使用して、各シャード内のデータをさら分割することができます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-146">For example, you might divide data into shards and then use vertical partitioning to further subdivide the data in each shard.</span></span>

<span data-ttu-id="f8a35-311">運用管理に影響する次の要因を考慮してください。</span><span class="sxs-lookup"><span data-stu-id="f8a35-311">Consider the following factors that affect operational management:</span></span>

* <span data-ttu-id="f8a35-317">**定期的にデータをアーカイブして削除する方法**。</span><span class="sxs-lookup"><span data-stu-id="f8a35-317">**How to archive and delete the data on a regular basis**.</span></span> <span data-ttu-id="f8a35-318">パーティションの過度な成長を防止するために、定期的に (たとえば、月単位で) データをアーカイブして削除する必要があります。</span><span class="sxs-lookup"><span data-stu-id="f8a35-318">To prevent the excessive growth of partitions, you need to archive and delete data on a regular basis (perhaps monthly).</span></span> <span data-ttu-id="f8a35-319">異なるアーカイブ スキーマに一致するように、データを変換することが必要な場合があります。</span><span class="sxs-lookup"><span data-stu-id="f8a35-319">It might be necessary to transform the data to match a different archive schema.</span></span>

## <a name="rebalancing-partitions"></a><span data-ttu-id="f8a35-323">パーティションの再調整</span><span class="sxs-lookup"><span data-stu-id="f8a35-323">Rebalancing partitions</span></span>

<span data-ttu-id="f8a35-324">システムが成熟するにつれて、パーティション分割構成を調整することが必要になる場合があります。</span><span class="sxs-lookup"><span data-stu-id="f8a35-324">As a system matures, you might have to adjust the partitioning scheme.</span></span> <span data-ttu-id="f8a35-325">たとえば、パーティション間でトラフィック量に不均衡が生じ、特定のパーティションがホットスポットになり、過度の競合が発生することがあります。</span><span class="sxs-lookup"><span data-stu-id="f8a35-325">For example, individual partitions might start get a disproportionate volume of traffic and become hot, leading to excessive contention.</span></span> <span data-ttu-id="f8a35-326">また、一部のパーティションでデータ量が過少に見積もられており、これらのパーティションが容量の上限に近づいていることもあります。</span><span class="sxs-lookup"><span data-stu-id="f8a35-326">Or you might have underestimated the volume of data in some partitions, causing some partitions to approach capacity limits.</span></span> 

<span data-ttu-id="f8a35-327">Cosmos DB などの一部のデータ ストアでは、パーティションを自動的に再調整できます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-327">Some data stores, such as Cosmos DB, can automatically rebalance partitions.</span></span> <span data-ttu-id="f8a35-328">それ以外の場合、再調整は次の 2 段階で構成される管理タスクとなります。</span><span class="sxs-lookup"><span data-stu-id="f8a35-328">In other cases, rebalancing is an administrative task that consists of two stages:</span></span>

1. <span data-ttu-id="f8a35-329">新しいパーティション分割戦略を決定します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-329">Determine a new partitioning strategy.</span></span> 

    - <span data-ttu-id="f8a35-330">どのパーティションを分割する (場合によっては結合する) 必要があるか。</span><span class="sxs-lookup"><span data-stu-id="f8a35-330">Which partitions need to be split (or possibly combined)?</span></span> 
    - <span data-ttu-id="f8a35-331">新しいパーティション キーは何か。</span><span class="sxs-lookup"><span data-stu-id="f8a35-331">What is the new partition key?</span></span>

2. <span data-ttu-id="f8a35-332">データを古いパーティション分割構成から新しいパーティション セットに移行します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-332">Migrate data from the old partitioning scheme to the new set of partitions.</span></span>

<span data-ttu-id="f8a35-333">データ ストアによっては、パーティションの使用中にパーティション間でデータを移行できる場合もあります。</span><span class="sxs-lookup"><span data-stu-id="f8a35-333">Depending on the data store, you might be able to migrate data between partitions while they are in use.</span></span> <span data-ttu-id="f8a35-334">これは "*オンライン移行*" と呼ばれます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-334">This is called *online migration*.</span></span> <span data-ttu-id="f8a35-335">これが不可能な場合は、データを再配置する間、パーティションを使用できないようにする必要があります ("*オフライン移行*")。</span><span class="sxs-lookup"><span data-stu-id="f8a35-335">If that's not possible, you might need to make partitions unavailable while the data is relocated (*offline migration*).</span></span>

### <a name="offline-migration"></a><span data-ttu-id="f8a35-336">オフライン移行</span><span class="sxs-lookup"><span data-stu-id="f8a35-336">Offline migration</span></span>

<span data-ttu-id="f8a35-337">オフライン移行は、競合が発生する可能性が低減されるので一般に簡単です。</span><span class="sxs-lookup"><span data-stu-id="f8a35-337">Offline migration is generally simpler, because it reduces the chances of contention occurring.</span></span> <span data-ttu-id="f8a35-338">概念的には、オフライン移行は次のように動作します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-338">Conceptually, offline migration works as follows:</span></span>

1. <span data-ttu-id="f8a35-339">パーティションをオフラインにします。</span><span class="sxs-lookup"><span data-stu-id="f8a35-339">Mark the partition offline.</span></span>
2. <span data-ttu-id="f8a35-340">データを分割/マージし、新しいパーティションに移動します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-340">Split-merge and move the data to the new partitions.</span></span>
3. <span data-ttu-id="f8a35-341">データを検証します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-341">Verify the data.</span></span>
4. <span data-ttu-id="f8a35-342">新しいパーティションをオンラインにします。</span><span class="sxs-lookup"><span data-stu-id="f8a35-342">Bring the new partitions online.</span></span>
5. <span data-ttu-id="f8a35-343">古いパーティションを削除します。</span><span class="sxs-lookup"><span data-stu-id="f8a35-343">Remove the old partition.</span></span>

<span data-ttu-id="f8a35-344">必要に応じて、手順 1. でパーティションを読み取り専用としてマークすることもできます。これにより、アプリケーションはデータの移動中も、データを読み取ることができます。</span><span class="sxs-lookup"><span data-stu-id="f8a35-344">Optionally, you can mark a partition as read-only in step 1, so that applications can still read the data while it is being moved.</span></span>

## <a name="online-migration"></a><span data-ttu-id="f8a35-345">オンライン移行</span><span class="sxs-lookup"><span data-stu-id="f8a35-345">Online migration</span></span>

<span data-ttu-id="f8a35-346">オンライン移行は実行が複雑になりますが、中断は少なくなります。</span><span class="sxs-lookup"><span data-stu-id="f8a35-346">Online migration is more complex to perform but less disruptive.</span></span> <span data-ttu-id="f8a35-347">プロセスはオフライン移行と似ていますが、元のパーティションがオフラインになることはありません。</span><span class="sxs-lookup"><span data-stu-id="f8a35-347">The process is similar to offline migration, except the original partition is not marked offline.</span></span> <span data-ttu-id="f8a35-348">移行プロセスの粒度 (たとえば、項目単位か、シャード単位か) に応じて、クライアント アプリケーションのデータ アクセス コードでは、2 つの場所 (元のパーティションと新しいパーティション) に保持されるデータの読み書きを処理することが必要になる場合があります。</span><span class="sxs-lookup"><span data-stu-id="f8a35-348">Depending on the granularity of the migration process (for example, item by item versus shard by shard), the data access code in the client applications might have to handle reading and writing data that's held in two locations, the original partition and the new partition.</span></span>
