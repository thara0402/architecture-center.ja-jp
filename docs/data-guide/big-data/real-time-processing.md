# <a name="real-time-processing"></a><span data-ttu-id="e1fa7-102">リアルタイム処理</span><span class="sxs-lookup"><span data-stu-id="e1fa7-102">Real time processing</span></span>

<span data-ttu-id="e1fa7-103">リアルタイム処理は、リアルタイムでキャプチャされるデータ ストリームを処理します。このデータは最小限の待機時間で処理されて、リアルタイム (または、ほぼリアルタイム) のレポートまたは自動化された応答が生成されます。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-103">Real time processing deals with streams of data that are captured in real-time and processed with minimal latency to generate real-time (or near-real-time) reports or automated responses.</span></span> <span data-ttu-id="e1fa7-104">たとえば、リアルタイム交通監視ソリューションでは、センサー データを使用して高い交通量を検出する場合があります。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-104">For example, a real-time traffic monitoring solution might use sensor data to detect high traffic volumes.</span></span> <span data-ttu-id="e1fa7-105">このデータを使用して、渋滞を表示する地図を動的に更新したり、相乗り車線やその他の交通管理システムを自動的に開始したりできます。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-105">This data could be used to dynamically update a map to show congestion, or automatically initiate high-occupancy lanes or other traffic management systems.</span></span>

![リアルタイム処理パイプラインの図](./images/real-time-pipeline.png)

<span data-ttu-id="e1fa7-107">リアルタイム処理は、とても短い処理待機時間要件 (ミリ秒または秒単位で測定) がある入力データのアンバウンド ストリームの処理として定義されています。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-107">Real-time processing is defined as the processing of unbounded stream of input data, with very short latency requirements for processing &mdash; measured in milliseconds or seconds.</span></span> <span data-ttu-id="e1fa7-108">一般にこの受信データは、JSON などの非構造化または半構造化形式で到着し、[バッチ処理](./batch-processing.md)と同じ処理要件を持ちますが、リアルタイムでの使用をサポートするためにターンアラウンド タイムは短くなります。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-108">This incoming data typically arrives in an unstructured or semi-structured format, such as JSON, and has the same processing requirements as [batch processing](./batch-processing.md), but with shorter turnaround times to support real-time consumption.</span></span>

<span data-ttu-id="e1fa7-109">多くの場合、処理されたデータは分析と視覚化のために最適化された分析データ ストアに書き込まれます。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-109">Processed data is often written to an analytical data store, which is optimized for analytics and visualization.</span></span> <span data-ttu-id="e1fa7-110">処理されたデータは、分析、ビジネス インテリジェンス、リアルタイム ダッシュボード視覚化のために分析およびレポート レイヤーに直接取り込むこともできます。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-110">The processed data can also be ingested directly into the analytics and reporting layer for analysis, business intelligence, and real-time dashboard visualization.</span></span>

## <a name="challenges"></a><span data-ttu-id="e1fa7-111">課題</span><span class="sxs-lookup"><span data-stu-id="e1fa7-111">Challenges</span></span>

<span data-ttu-id="e1fa7-112">リアルタイム処理ソリューションの大きな課題の 1 つは、メッセージの取り込み、処理、格納を (特に大量のメッセージにおいて) リアルタイムで行うことです。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-112">One of the big challenges of real-time processing solutions is to ingest, process, and store messages in real time, especially at high volumes.</span></span> <span data-ttu-id="e1fa7-113">処理は、取り込みパイプラインをブロックしないように行う必要があります。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-113">Processing must be done in such a way that it does not block the ingestion pipeline.</span></span> <span data-ttu-id="e1fa7-114">データ ストアでは、大量の書き込みをサポートする必要があります。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-114">The data store must support high-volume writes.</span></span> <span data-ttu-id="e1fa7-115">もう 1 つの課題は、リアルタイムでのアラートの生成やリアルタイム (または、ほぼリアルタイム) のダッシュボードでのデータの表示など、データをすばやく操作できることです。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-115">Another challenge is being able to act on the data quickly, such as generating alerts in real time or presenting the data in a real-time (or near-real-time) dashboard.</span></span>

## <a name="architecture"></a><span data-ttu-id="e1fa7-116">アーキテクチャ</span><span class="sxs-lookup"><span data-stu-id="e1fa7-116">Architecture</span></span>

<span data-ttu-id="e1fa7-117">リアルタイム処理アーキテクチャには、次の論理コンポーネントがあります。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-117">A real-time processing architecture has the following logical components.</span></span>

- <span data-ttu-id="e1fa7-118">**リアルタイム メッセージ取り込み。**</span><span class="sxs-lookup"><span data-stu-id="e1fa7-118">**Real-time message ingestion.**</span></span> <span data-ttu-id="e1fa7-119">アーキテクチャには、ストリーム処理コンシューマーが使用するリアルタイム メッセージをキャプチャして格納する方法が含まれている必要があります。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-119">The architecture must include a way to capture and store real-time messages to be consumed by a stream processing consumer.</span></span> <span data-ttu-id="e1fa7-120">シンプルなケースでは、このサービスは、新しいメッセージがフォルダーに格納される単純なデータ ストアとして実装できます。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-120">In simple cases, this service could be implemented as a simple data store in which new messages are deposited in a folder.</span></span> <span data-ttu-id="e1fa7-121">しかし多くの場合、ソリューションには、メッセージのバッファーとして機能する Azure Event Hubs などのメッセージ ブローカーが必要です。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-121">But often the solution requires a message broker, such as Azure Event Hubs, that acts as a buffer for the messages.</span></span> <span data-ttu-id="e1fa7-122">メッセージ ブローカーでは、スケールアウト処理と信頼性の高い配信がサポートされる必要があります。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-122">The message broker should support scale-out processing and reliable delivery.</span></span>

- <span data-ttu-id="e1fa7-123">**ストリーム処理。**</span><span class="sxs-lookup"><span data-stu-id="e1fa7-123">**Stream processing.**</span></span> <span data-ttu-id="e1fa7-124">このソリューションでは、リアルタイム メッセージを取得した後、分析用にデータをフィルターしたり、集計したり、その他の準備を行ったりして、それらのメッセージを処理する必要があります。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-124">After capturing real-time messages, the solution must process them by filtering, aggregating, and otherwise preparing the data for analysis.</span></span>

- <span data-ttu-id="e1fa7-125">**分析データ ストア。**</span><span class="sxs-lookup"><span data-stu-id="e1fa7-125">**Analytical data store.**</span></span> <span data-ttu-id="e1fa7-126">多くのビッグ データ ソリューションは、分析用にデータを準備した後、処理されたデータを分析ツールを使用してクエリを実行できる構造化された形式で提供するように設計されています。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-126">Many big data solutions are designed to prepare data for analysis and then serve the processed data in a structured format that can be queried using analytical tools.</span></span>

- <span data-ttu-id="e1fa7-127">**分析とレポート。**</span><span class="sxs-lookup"><span data-stu-id="e1fa7-127">**Analysis and reporting.**</span></span> <span data-ttu-id="e1fa7-128">ほとんどのビッグ データ ソリューションの目的は、分析とレポートによってデータに関する実用的な情報を提供することにあります。</span><span class="sxs-lookup"><span data-stu-id="e1fa7-128">The goal of most big data solutions is to provide insights into the data through analysis and reporting.</span></span>
